Multi-User Access Redesign

Redesigning one of Eventbrite’s most broken features

Overview

Eventbrite is the world’s largest marketplace for live experiences like concerts, food festivals, and marathons. The multi-user access feature allows event organizers to manage and collaborate on events together. The feature was incredibly broken long overdue for a redesign.

My Team & Role

I was the only User Experience Designer on the team for this project. The team also had a Product Manager, and three backend engineers. I was responsible for concept sketches, wireframes, user flows, and user testing.

Invision’s Collaboration Feature

Research

During this phase of the project I interviewed stakeholders, the support team and account managers to identify all the pain points with the existing design. I also explored products that had a similar collaboration feature for inspiration.

Ideation

I identified solutions that would solve the feature’s problems I discovered in the research phase.

1. Dependencies of “sub-user and parent users” needed to be removed.
2. Organizers need to be able to invite already-existing Eventbrite users to collaborate on an event.
3. All organizers should be able to create their own events.
4. Permissions should be grouped into roles for simplification.
5. Permissions should be reworded for clarity.
6. The feature should be more visible and scale better.
7. Content should be disabled not hidden when a user does not have access.

User Testing

We had little time to test during this project, so I chose to test the best interaction for editing a collaborator’s custom permissions. There were three main concepts I tested: expandable drawers, dropdowns, and an edit button.

Current Status

After ending my internship, the project moved into the product design and engineering phases. The feature is scheduled to release early this year.
